.. Title

cisco.dnac.transit_peer_network_v1 module -- Resource module for Transit Peer Network V1
++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `cisco.dnac collection <https://galaxy.ansible.com/cisco/dnac>`_ (version 6.30.2).

    To install it, use: :code:`ansible-galaxy collection install cisco.dnac`.
    You need further requirements to be able to use this module,
    see :ref:`Requirements <ansible_collections.cisco.dnac.transit_peer_network_v1_module_requirements>` for details.

    To use it in a playbook, specify: :code:`cisco.dnac.transit_peer_network_v1`.

Synopsis
--------

.. Description

- Manage operations create and delete of the resource Transit Peer Network V1.
- Add Transit Peer Network in SD-Access.
- Delete Transit Peer Network from SD-Access.

.. note::
    This module has a corresponding :ref:`action plugin <action_plugins>`.

Requirements
------------
The below requirements are needed on the host that executes this module.

- dnacentersdk \>= 2.4.9
- python \>= 3.5

.. Notes

Notes
-----

.. note::
   - SDK Method used are sda.Sda.add\_transit\_peer\_network\_v1, sda.Sda.delete\_transit\_peer\_network\_v1,
   - Paths used are post /dna/intent/api/v1/business/sda/transit-peer-network, delete /dna/intent/api/v1/business/sda/transit-peer-network,
   - Does not support \ :literal:`check\_mode`\ 
   - The plugin runs on the control node and does not use any ansible connection plugins, but instead the embedded connection manager from Cisco DNAC SDK
   - The parameters starting with dnac\_ are used by the Cisco DNAC Python SDK to establish the connection

.. Examples

Examples
--------

.. code-block:: yaml+jinja

    
    - name: Delete all
      cisco.dnac.transit_peer_network_v1:
        dnac_host: "{{dnac_host}}"
        dnac_username: "{{dnac_username}}"
        dnac_password: "{{dnac_password}}"
        dnac_verify: "{{dnac_verify}}"
        dnac_port: "{{dnac_port}}"
        dnac_version: "{{dnac_version}}"
        dnac_debug: "{{dnac_debug}}"
        state: absent
        transitPeerNetworkName: string

    - name: Create
      cisco.dnac.transit_peer_network_v1:
        dnac_host: "{{dnac_host}}"
        dnac_username: "{{dnac_username}}"
        dnac_password: "{{dnac_password}}"
        dnac_verify: "{{dnac_verify}}"
        dnac_port: "{{dnac_port}}"
        dnac_version: "{{dnac_version}}"
        dnac_debug: "{{dnac_debug}}"
        state: present
        ipTransitSettings:
          autonomousSystemNumber: string
          routingProtocolName: string
        sdaTransitSettings:
          transitControlPlaneSettings:
          - deviceManagementIpAddress: string
            siteNameHierarchy: string
        transitPeerNetworkName: string
        transitPeerNetworkType: string

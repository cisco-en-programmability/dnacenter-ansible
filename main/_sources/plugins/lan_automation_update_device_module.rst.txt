cisco.dnac.lan_automation_update_device module -- Resource module for Lan Automation Updatedevice
+++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This module is part of the `cisco.dnac collection <https://galaxy.ansible.com/cisco/dnac>`_ (version 6.22.0).

    To install it, use: :code:`ansible-galaxy collection install cisco.dnac`.

    see :ref:`Requirements <ansible_collections.cisco.dnac.lan_automation_update_device_module_requirements>` for details.

Synopsis
--------

.. Description

- Manage operation update of the resource Lan Automation Updatedevice.
- Invoke this API to perform a DAY-N update on LAN Automation-related devices. Supported features include Loopback0 IP update, hostname update, link addition, and link deletion.


.. note::
    This module has a corresponding :ref:`action plugin <action_plugins>`.

------------
The below requirements are needed on the host that executes this module.

- python \>= 3.5

.. Notes

Notes
-----

.. note::
   - SDK Method used are lan\_automation.LanAutomation.lan\_automation\_device\_update,
   - Paths used are put /dna/intent/api/v1/lan-automation/updateDevice,
   - Does not support \ :literal:`check\_mode`\ 
   - The plugin runs on the control node and does not use any ansible connection plugins, but instead the embedded connection manager from Cisco DNAC SDK
   - The parameters starting with dnac\_ are used by the Cisco DNAC Python SDK to establish the connection

.. Examples

Examples
--------

.. code-block:: yaml+jinja

    
    - name: Update all
      cisco.dnac.lan_automation_updateDevice:
        dnac_host: "{{dnac_host}}"
        dnac_username: "{{dnac_username}}"
        dnac_password: "{{dnac_password}}"
        dnac_verify: "{{dnac_verify}}"
        dnac_port: "{{dnac_port}}"
        dnac_version: "{{dnac_version}}"
        dnac_debug: "{{dnac_debug}}"
        feature: string
        hostnameUpdateDevices:
        - deviceManagementIPAddress: string
          newHostName: string
        linkUpdate:
          destinationDeviceInterfaceName: string
          destinationDeviceManagementIPAddress: string
          ipPoolName: string
          sourceDeviceInterfaceName: string
          sourceDeviceManagementIPAddress: string
        loopbackUpdateDeviceList:
        - deviceManagementIPAddress: string
          newLoopback0IPAddress: string
